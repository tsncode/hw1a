Homework 1: Tasks for sm1248

Your personal task is to develop a system for apartments and their tenants.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2123.hw1.domain.Tenant. With the following private fields:

  1. String name
  2. String department
  3. String role
  4. int contract
Generate necessary getters and setters. [7 marks]

Create class co2123.hw1.domain.Apartment. With the following private fields:

  1. int id
  2. String address
  3. String type
  4. List<Tenant> tenants
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2123.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Apartment> apartments in class co2123.hw1.Hw1Application. Initialize the list inside class co2123.hw1.Hw1Application to contain one apartment with two tenants. [4 marks]

Create controller co2123.hw1.controller.ApartmentController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/apartments" should show view apartments/list. [2 marks]
    • Add the list of all apartments from co2123.hw1.Hw1Application to the model as attribute apartments. [2 marks]
    • Create the corresponding JSP to list the address and type of each apartment followed by a link to URL "/tenants?apartment={id of apartment}", e.g., "/tenants?apartment=3". [4 marks]
  2. Requests for URL "/newApartment" should show view apartments/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="apartment" and action="/addApartment" and inputs for all fields of class Apartment except for tenants. [5 marks]
  3. POST requests for URL "/addApartment" should add a new apartment to the list of all apartments in co2123.hw1.Hw1Application and redirect to /apartments. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2123.hw1.controller.TenantController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/tenants" should show view tenants/list. [2 marks]
    • Based on request parameter apartment determine the corresponding apartment from co2123.hw1.Hw1Application. [2 marks]
    • Add the list of all tenants of that apartment to the model as attribute tenants. [2 marks]
    • Create the corresponding JSP for the view to list the name, department, role, and contract of each tenant. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newTenant?apartment={id of apartment whose tenants are listed}", e.g., "/newTenant?apartment=3". [4 marks]
  2. Requests for URL "/newTenant" should show view tenants/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="tenant" and action="/addTenant" and inputs for all fields of class Tenant. [6 marks]
    • Also add the request parameter apartment with the ID of the current apartment. [4 marks]
  3. POST requests for URL "/addTenant" should add a new tenant to the list of tenants of the apartment identified by request parameter apartment and then redirect to /apartments. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Apartment and make it available in controller ApartmentController. [4 marks]

When adding a new apartment validate that

  1. The value of field id is unique within the list Hw1Application.apartments. [3 marks]
  2. The value of field address is not empty. [2 marks]
  3. The value of field type is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Tenant and make it available in controller TenantController. [4 marks]

When adding a new tenant validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field department is not empty. [2 marks]
  3. The value of field role is either "Undergraduate", "Postgraduate", or "Staff". [3 marks]
  4. The value of field contract is between (including) 21 and 42. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
